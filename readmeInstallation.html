Welcome to phpScheduleIt 2.0! A quick A),B),C) installation.

This is a pre-release version of phpScheduleIt. It will certainly contain bugs and is not yet suitable for production use.

Prerequisite

Note: for users without web host service, or existing environment.
If you want to accomplish all of the below quick and easy, try this apachefriends.org, it will do it for you in no time. XAMMP (highly recommended).
  • PHP 5.2 or greater.
  • MySQL 5.0 or greater.
  • Web server (e.g Apache2, etc...).
  • OS - Windows, Mac, Unix-like were tested.

A) Application Deployment to Server

  • Untar or unzip the archived distribution to your webserver's document root. Or
  • If you don't have direct access to your document root or use hosting service, then transfer the extracted directory to your web server's document root using, for example, ftp or WinSCP.
  • Important! The web server must have write access to /your-phpScheduleIt/tpl_c and /your-phpScheduleIt/tpl (want to know why?).

B) Application Configuration

You can configure phpScheduleIt to fit your environments and needs, or use the minimal default settings which should be enough for the application to work. We recommend you to change according to your specifics.
  • In addition, to allow resource image uploads, the web-server must also have read/write access to your configurable uploads directory of "$conf['settings']['image.upload.directory']" in the config.php.
  • To configure the application, you can open /config/config.php and alter any settings accordingly. The admin email address can be set in the phpScheduleIt/config/config.php file of setting $conf['settings']['admin.email'].
    When later register an account with admin email address the user will be given admin privilege.
  • An alternative to the default email authentication method is LDAP (this is optional). To configure LDAP authentication, you can open /config/config.php and alter any settings accordingly. Examples are shown next to the parameters.
Note: If you try to load the application at this time (eg. http://yourhostname/phpScheduleIt/Web/), you will probably get a white page. This is because there is no backend database configured yet. So continue on ...

C) Database Setup

You have 2 ways to set up your database for the application to work.

1) Automatically

The packaged database scripts make assumptions about your desired database configuration and set default values. Please edit them to suit your environment before running. And the files are located in phpScheduleIt/database_schema/.
  • The automated database setup only supports MySQL at this time. To run the automated database setup, navigate to the /Web/install directory in a web browser and follow the on-screen instructions.
  • Note: Some may see directory permission issues displayed on the page. Recall that (The web server must have write access to /your-phpScheduleIt/tpl_c and /your-phpScheduleIt/tpl). If you cannot provide the required permission.
    Contact your web server administrator or hosting service to resolve.
OR

2) Manually

The packaged database scripts make assumptions about your desired database configuration and set default values. Please edit them to suit your environment before running. And the file are located in phpScheduleIt/database_schema/.
Go ahead import the following sql files in the listed order (recommending phpmyadmin interface):
  • Note: Some may see directory permission issues during the following process. Recall that (The web server must have write access to /your-phpScheduleIt/tpl_c and /your-phpScheduleIt/tpl). If you cannot provide the required permission.
    Contact your web server administrator or hosting service to resolve.
  • import "/database_schema/create-db.sql" to mysql db (phpscheduleit2 db created)
  • import "/database_schema/schema-utf8.sql" to phpscheduleit2 (db schema created)
  • import "/database_schema/create-user.sql" to mysql db (schedule_user/password mysql account created to match to your default configuration and notice: this is not the application user!).
    At this point with the aforementioned default configuration in your-phpScheduleIt/config/config.php your application can connect to the database and loaded eg. see http://localhost/your-phpScheduleIt/Web/index.php
    but you can not login since there is no application data or user, so go to next step
  • import "/database_schema/data-utf8.sql" to phpscheduleit2 (initializing application database)
  • import "/database_schema/testdata-utf8.sql" to phpscheduleit2 (a sample application data created with 2 users: 1) admin/password and 2) nkorbel/password). Use these users for testing your installation. You are done. Congratulation!
    Try to load the application at (eg. http://yourhostname/phpScheduleIt/Web/).

The First Login

There are 2 main types of account, they are admin and user account.
  • Aforementioned if you imported a sample application data, you now can use 1) admin/password and 2) nkorbel/password to login and make changes or addition via the application.
  • Or you will need to register for an account with your configured admin email address. The admin email address can be set in the phpScheduleIt/config/config.php file of setting $conf['settings']['admin.email'].
    Other self registration accounts are defaulted to normal users.
    After a shelf registration, you will be logged in automatically. Important! At this time, you are recommended to change your password immediately. Because in the case your mail server fails to send you your login information you have already changed the password.
    Notice that an email address is required to register for an account.
  • For the email registration process to work (it will send you an email with account login information). You need to make sure your email server is working and the application is configured correctly. Look for smtp.* parameters in phpScheduleIt/config/config.php,
    and fill them in with your mail server information. Examples are shown next to the parameters.
  • For LDAP authentication please login with your LDAP username/password.

Log Files

phpScheduleIt logs multiple levels of information categorized into either application or database logs. To do this, the log4php library is used. By default, logging is turned OFF.
  • First, rename /config/log4php.config.dist.xml to /config/log4php.config.xml
  • To allow application logging, the PHP account requires read/write access to your configured log directory
  • Logging is configured in /config/log4php.config.xml
  • Logging is controlled by changing the <level> of each <logger>
  • Levels used by phpScheduleIt are OFF, DEBUG, ERROR. For normal operation, ERROR is appropriate. If trace logs are needed, DEBUG is appropriate.
  • To turn on application logging, change the <level value="OFF" /> to an appropriate level for either the default or sql loggers. For example, <level value="ERROR" />
  • For more information on logging configuration, visit Log4Php
